СВОДКА ПРОЕКТА «ЭЛЕКТРОННЫЙ ДНЕВНИК ДЛЯ ШКОЛЫ»
=================================================

1. ОБЩЕЕ ОПИСАНИЕ
==================

Веб-приложение «Электронный дневник» — это информационная система
для автоматизации учебного процесса в школе. Система обеспечивает
электронный учёт успеваемости, посещаемости, расписания уроков,
а также внутреннюю коммуникацию между участниками образовательного
процесса (администрация, учителя, ученики, родители).

Приложение разработано на языке PHP без использования фреймворков
с применением архитектурного паттерна MVC (Model-View-Controller),
СУБД MySQL, и современного интерфейса на базе Tailwind CSS.


2. ЦЕЛЬ И ЗАДАЧИ ПРОЕКТА
==========================

Цель: разработка веб-приложения для автоматизации ведения 
школьного дневника и журнала успеваемости.

Задачи:
- Реализовать систему авторизации с ролевой моделью доступа (7 ролей)
- Разработать модуль управления пользователями (CRUD-операции)
- Создать электронный журнал оценок с интерактивным интерфейсом
- Реализовать модуль учёта посещаемости с 4 статусами
- Разработать модуль расписания уроков с редактором
- Создать модуль отчётов и аналитики с визуализацией (графики)
- Реализовать систему внутренних сообщений
- Обеспечить безопасность: защита от SQL-инъекций, XSS, CSRF, брутфорса


3. ТЕХНОЛОГИЧЕСКИЙ СТЕК
========================

Серверная часть (Backend):
- Язык: PHP 7.2+
- База данных: MySQL 5.7+ / 8.0
- Взаимодействие с БД: PDO с подготовленными запросами (prepared statements)
- Архитектура: MVC (Model-View-Controller) без фреймворка
- Хеширование паролей: password_hash() с алгоритмом bcrypt
- Управление сессиями: PHP sessions с регенерацией ID

Клиентская часть (Frontend):
- HTML5, CSS3, JavaScript (ES6)
- CSS-фреймворк: Tailwind CSS 3.x (подключён через CDN)
- Иконки: Font Awesome 6.x
- Графики: Chart.js 4.x
- AJAX-запросы: Fetch API

Среда разработки и развёртывания:
- Open Server Panel 5.4.3 (Apache 2.4 + PHP + MySQL)


4. РОЛЕВАЯ МОДЕЛЬ (7 ролей)
=============================

4.1. Администратор (admin)
    - Полный доступ ко всем модулям системы
    - Управление пользователями: создание, редактирование, удаление
    - Управление классами и предметами
    - Назначение ролей и привязка данных (учитель→предметы, родитель→дети)
    - Просмотр всех данных и отчётов

4.2. Директор (director)
    - Просмотр общей статистики по школе
    - Доступ ко всем отчётам (успеваемость, посещаемость, нагрузка)
    - Просмотр списка пользователей (без редактирования)
    - Просмотр журнала оценок и расписания
    - Отправка сообщений всем пользователям

4.3. Завуч (head_teacher)
    - Управление расписанием: добавление, удаление, копирование
    - Контроль успеваемости и посещаемости по всем классам
    - Формирование отчётов
    - Просмотр нагрузки учителей

4.4. Классный руководитель (class_teacher)
    - Доступ к журналу своего класса по всем предметам
    - Просмотр посещаемости учеников своего класса
    - Выставление оценок по своим предметам
    - Отправка сообщений родителям учеников своего класса
    - Формирование отчётов по классу

4.5. Учитель (teacher)
    - Выставление оценок только по своим предметам и в своих классах
    - Отметка посещаемости на своих уроках
    - Просмотр своего расписания
    - Обмен сообщениями с коллегами и родителями

4.6. Ученик (student)
    - Просмотр своих оценок с группировкой по предметам
    - Просмотр средних баллов и графиков успеваемости
    - Просмотр своей посещаемости со статистикой
    - Просмотр расписания своего класса
    - Отправка сообщений учителям

4.7. Родитель (parent)
    - Просмотр оценок своих детей (поддержка нескольких детей)
    - Просмотр посещаемости детей
    - Просмотр расписания класса ребёнка
    - Обмен сообщениями с учителями


5. ФУНКЦИОНАЛЬНЫЕ МОДУЛИ
=========================

5.1. Модуль авторизации
    - Форма входа (логин + пароль)
    - Защита от брутфорса: ограничение попыток входа (5 попыток,
      блокировка на 15 минут), учёт по IP-адресу и логину
    - Хеширование паролей через bcrypt (password_hash / password_verify)
    - Регенерация ID сессии после входа (защита от session fixation)
    - Таймаут сессии (автоматический выход через 1 час неактивности)

5.2. Модуль управления пользователями
    - Список пользователей с фильтрацией по ролям и поиском
    - Создание пользователя с динамическими полями в зависимости от роли:
      * Для ученика: выбор класса
      * Для учителя: назначение предметов и классов
      * Для родителя: привязка детей с указанием типа родства
      * Для классного руководителя: назначение класса
    - Редактирование и удаление пользователей
    - Управление классами: создание, удаление
    - Управление предметами: создание, удаление
    - Пагинация списка пользователей

5.3. Модуль журнала оценок
    - Интерактивный журнал: таблица "ученики × даты"
    - Фильтрация по предмету, классу, периоду дат
    - Добавление оценок через модальное окно (AJAX, без перезагрузки)
    - Редактирование и удаление оценок через модальное окно
    - Типы оценок: текущая, домашняя работа, контрольная, экзамен
    - Комментарии к оценкам
    - Автоматический расчёт среднего балла по каждому ученику
    - Цветовое кодирование: 5=зелёный, 4=синий, 3=жёлтый, 2=красный
    - Разграничение прав: учитель ставит оценки только по своим предметам
    - Для учеников и родителей: просмотр оценок с графиками Chart.js

5.4. Модуль посещаемости
    - Отметка посещаемости по классу, предмету и дате
    - 4 статуса: присутствует, отсутствует, опоздал, уважительная причина
    - Цветовая индикация статусов
    - Массовая отметка «Все присутствуют»
    - Комментарии к записям посещаемости
    - Статистика для учеников: проценты, диаграмма (Chart.js)
    - Группировка статистики по предметам

5.5. Модуль расписания
    - Просмотр расписания: сетка по дням недели (Пн-Сб)
    - Подсветка текущего урока и текущего дня
    - Для учителя: персональное расписание (свои уроки)
    - Для ученика/родителя: расписание класса
    - Редактор расписания (для завуча/админа):
      * Визуальная таблица урок×день с кликабельными ячейками
      * Добавление уроков через модальное окно
      * Удаление уроков наведением мыши
      * Автоподстановка времени по номеру урока
      * Проверка на пересечения (один учитель не может вести 
        два урока одновременно)
      * Копирование расписания из одного класса в другой

5.6. Модуль отчётов и аналитики
    - Отчёт по успеваемости класса:
      * Средний балл по каждому предмету (столбчатая диаграмма)
      * Распределение оценок (круговая диаграмма)
      * Рейтинг учеников с медалями (🥇🥈🥉)
      * Подсчёт отличников, хорошистов, качества знаний (%)
    - Отчёт по посещаемости:
      * Процентное соотношение статусов (круговая диаграмма)
      * Таблица посещаемости по ученикам с прогресс-барами
      * Список часто пропускающих учеников
    - Итоговая ведомость:
      * Сводная таблица: ученики × предметы × средние баллы
      * Функция печати через window.print()
    - Нагрузка учителей:
      * Столбчатая диаграмма часов по учителям
      * Детализация: предметы, классы, часы в неделю
      * Цветовая индикация нагрузки (норма/повышенная)
    - Профиль ученика:
      * Карточка с информацией (класс, родители, контакты)
      * Средние баллы по предметам (график + таблица)
      * Статистика посещаемости
      * Последние оценки

5.7. Модуль сообщений
    - Внутренняя почта между пользователями
    - Входящие и отправленные сообщения
    - Форма написания нового сообщения с выбором получателя
    - Ответ на сообщение (цитирование оригинала)
    - Отметка о прочтении (дата и время)
    - Счётчик непрочитанных сообщений (в меню и шапке)
    - Разграничение получателей по ролям:
      * Админ/директор → все пользователи
      * Учитель → коллеги, родители, завуч, директор
      * Родитель → учителя, классный руководитель
      * Ученик → учителя
    - Автоматическое обновление счётчика (AJAX, каждые 30 сек)


6. СТРУКТУРА БАЗЫ ДАННЫХ
==========================

Всего 17 таблиц:

6.1. roles — роли пользователей (7 записей)
6.2. users — пользователи системы
     Поля: id, login, password_hash, full_name, email, phone,
           role_id (FK→roles), is_active, last_login, created_at, updated_at

6.3. subjects — учебные предметы
6.4. teachers — учителя (связь с users: user_id)
6.5. classes — классы (name, year, class_teacher_id→teachers)
6.6. students — ученики (user_id→users, class_id→classes)
6.7. parent_student — связь родитель↔ребёнок (M:N)
     Поля: parent_user_id, student_id, relationship, is_primary

6.8. teacher_subjects — предметы учителя (M:N)
6.9. teacher_class_subjects — предметы учителя по классам (M:N:N)
6.10. teacher_workload — нагрузка учителя (часы в неделю)

6.11. grades — оценки
      Поля: student_id, subject_id, teacher_id, grade(1-5),
            date, comment, grade_type, created_at

6.12. attendance — посещаемость
      Поля: student_id, subject_id, date,
            status(present/absent/late/excused), comment, marked_by

6.13. schedule — расписание уроков
      Поля: class_id, subject_id, teacher_id, day_of_week(1-6),
            lesson_order(1-8), time_start, time_end, room

6.14. terms — учебные четверти/семестры
6.15. final_grades — итоговые оценки за четверть
6.16. messages — внутренние сообщения
6.17. login_attempts — журнал попыток входа (защита от брутфорса)

Все внешние ключи используют каскадное удаление (ON DELETE CASCADE)
или ограничение (ON DELETE RESTRICT) в зависимости от логики.

8. МЕРЫ БЕЗОПАСНОСТИ
=====================

8.1. Защита от SQL-инъекций:
     Все запросы через PDO с подготовленными выражениями (prepare/execute)

8.2. Защита от XSS:
     Экранирование вывода через htmlspecialchars() (функция e())

8.3. Защита от CSRF:
     Токены в каждой форме, проверка при обработке POST-запросов

8.4. Защита от брутфорса:
     Таблица login_attempts, ограничение 5 попыток за 15 минут

8.5. Безопасное хранение паролей:
     bcrypt через password_hash() / password_verify()

8.6. Управление сессиями:
     Регенерация session_id после входа, таймаут неактивности,
     httponly cookies

8.7. Контроль доступа (RBAC):
     Middleware-функции requireAuth(), requireRole() в каждом контроллере
